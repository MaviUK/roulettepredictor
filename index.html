Roulette Directional Bias Predictor

Spins are assumed to alternate direction (CW, CCW, CW, CCW, ...). For each spin it records the wheel distance travelled in that spin's direction. To predict the next spin, it uses a Bayesian Bias Detection (Dirichlet–Multinomial) model:
• Uses the last directional distances in the upcoming direction,
• Applies a Dirichlet prior over all 37 possible distances,
• Updates that prior with recency-weighted evidence,
• Picks the distance with the highest posterior probability as the next bias,
• Then adds an optional extra pocket shift,
• CSV optimiser scans parameters to maximise hit rate,
• A Martingale on stake per number resets after 11 consecutive losses.

Base = starting stake per number for Live, Simulator, and CSV runs. Martingale doubles this after each losing bet, and resets after 11 consecutive losses.

Current model parameters: using defaults (extraShift=0, alpha=0.35, recency=0.97, lookback=20). Upload and optimise from CSV to auto-tune these for best accuracy.
